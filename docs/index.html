This API only has one route: one that creates a firework.

Make a request, using fetch, to the RESTful route you'd use to create a firework. If you need to refresh your memory on REST, consult a reference of your choosing.

Unlike a GET, making a POST, PUT, PATCH, or DELETE request requires passing a second argument to fetch. Consult the MDN documentation for fetch for more details.

You'll need to:

  1. Specify the request method (fetch defaults to GET)
  2. Provide stringified JSON as the body for the request
  3. Use a Content-Type header so that the API knows what kind of data you're sending. (Bonus points for an Accepts header, but make sure you know the difference).
Consult the MDN documentation for fetch to learn more about why and how to do these.

If you get a 422 error, that's good news! You're on the right track. Consult the network tab or parse the JSON that the API returned for more information about the error.

On a Mac, the keyboard shortcut to get an emoji picker is control-command-space.

Good luck!
